3.2. customizing our site

N

ext, we have to modify the example site to adjust it to our own requirements and introduce our own contents instead of the existing one. Freshly unpacked files represent an example site containing these instructions we are currently reading. There are several files and folders of our interest:

  • index.html - this is the main file containing social media promotional informations, visible project title and a list of SEO keywords.
  • orbital.html - this file usually won't need to be referenced unless we want override the default index.html interface and use our own layout, placing the orbital.html reference inside custom iframe structures as a part of user defined index.html.
  • orbital.json - in this file we can setup exposed E-fun rendering parameters.
  • contents/ - this is a folder where we keep our site content files. contents/contents.xml is the main XML file where we structure the site contents. The site content files (HTML, CSS, images, ...) should all be copied under the same folder and referred from contents/contents.xml file. Actually, the content files may be structured in any folders, but it is recommended to place the entire structure under contents/ folder. That would keep our site clean and tidy.

Hopefully, examining the default site example (that is, these instructions) should be descriptive enough to compose our own site. This is the quirky part where we will be using file manager and text editor to arrange and edit files of our interst. Please refer to provided links if you need to get more familiar with mentioned notions. This procedure is what makes E-fun site creating experience "slack", and requires some more advanced knowledge from the site creators.

Upon each edit, we would want to test if changes we made reflect what we want them to be. Again, we can test them by entering the local HTTP server address into a web browser.
